mybatis介绍:

前生是ibatis，它封装了JDBC的持久层框架，提供了对数据库操作的接口：SqlSessionFactoryBuilder SqlSessionFactory SqlSession  
它是半ORM框架（类和表的映射）

mybatis和Spring整合

步骤

1.导jar包：

spring-jdb:4.2.4.RELEASE 
mysql-connector-java:5.1.32
mybatis:3.2.8
mybatis-spring:1.2.2

2.配置文件

<1> 在applicationContext-*.xml 文件中

 <?xml version="1.0" encoding="UTF-8"?>
 <beans xmlns="http://www.springframework.org/schema/beans"
 xmlns:context="http://www.springframework.org/schema/context" xmlns:p="http://www.springframework.org/schema/p"
 xmlns:aop="http://www.springframework.org/schema/aop" xmlns:tx="http://www.springframework.org/schema/tx"

 http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd
 http://www.springframework.org/schema/aop http://www.springframework.org/schema/aop/spring-aop.xsd 
 http://www.springframework.org/schema/tx http://www.springframework.org/schema/tx/spring-tx.xsd
 http://www.springframework.org/schema/util http://www.springframework.org/schema/util/spring-util.xsd">

    <!-- 数据库连接池 -->
    <!-- 加载配置文件 -->
      <context:property-placeholder location="classpath*:properties/*.properties" />
    <!-- 数据库连接池 jdbc dbcp alibaba等类型-->
      <bean id="dataSource" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
        <property name="url" value="${jdbc.url}" />
        <property name="username" value="${jdbc.username}" />
        <property name="password" value="${jdbc.password}" />
        <property name="driverClassName" value="${jdbc.driver}" />
       </bean>
     <!-- 让spring管理sqlsessionfactory 使用mybatis和spring整合包中的 -->
      <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
     <!-- 数据库连接池 -->
       <property name="dataSource" ref="dataSource" />
      <!-- 加载mybatis的全局配置文件 -->
      </bean>
      <bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
      <property name="basePackage" value="com" />
 </bean>

  <2> 在 properties/*.properties

3.建立映射接口和代理mapper

(1)接口：
   public interface UserMapper {
    //查询所有，返回map类型
   List<Map<String, Object>> findAll();
   }
 (2)*mapper.xml:
    <?xml version="1.0" encoding="UTF-8" ?>
   <!DOCTYPE mapper    
   PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"    
   "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
   <mapper namespace="com.mapper.UserMapper">
    <select id="findAll" resultType="Map">
      select * from user
    </select>
   </mapper>

git

git分布式的版本控制

  • git是由Linux开发的，所以git和Linux文件系统结合的比较紧密。
  • git中没有中央版本库的说法，但是为了开发小组的代码共享，通常会搭建一个远程仓库，从某种程度说本地仓库和远程仓库在身份上是等价的

    本地到远程的步骤

    1. 本地创建一个git库，并将其add到远程git库中
    2. 在本地添加或者删除文件，然后commit提交到本地的git库中
    3. 将本地git库的分支push到远程git库的分支，如果这个时候远程git库中已经有别人push过，那么远程git库将不允许push，这时候需要先pull，然后如果有冲突，处理好冲突，commit到本地git仓库,再push到远程git库

svn

svn集中式版本管理控制

* 具有中央版本库
* 自动备份功能
